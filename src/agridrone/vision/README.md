# vision/ — AI / ML Inference Pipeline

This folder contains everything related to running the **3-model ensemble** and
producing a final disease prediction from an image.

## How the Pipeline Works

```
Image
  │
  ▼
[YOLOv8n-cls]  ──────────────────────────────┐
  │  top-5 predictions + confidence scores    │
  ▼                                           ▼
[Rule Engine]  (agronomic rules, monitor only)  [LLaVA via Ollama]
  │                                           │  natural-language visual check
  └──────────────────┬────────────────────────┘
                     ▼
            [Ensemble Voter]
            60% LLaVA  +  40% YOLO
                     │
                     ▼
            Final Prediction + Confidence
```

## Files

| File | Role |
|------|------|
| `ensemble_voter.py` | **Main combiner** — merges YOLO + LLaVA + rules into one answer |
| `disease_reasoning.py` | **Orchestrator** — calls each model and passes results to voter |
| `rule_engine.py` | Agronomic rules scoring (currently monitor-only, not overriding) |
| `rule_engine_base.py` | Base class / shared logic for rule engines |
| `rules_learned.py` | Rules learned from training data (auto-generated) |
| `rules_llm.py` | Rules generated by LLM reflection |
| `llm_validator.py` | Sends image to Ollama/LLaVA, returns structured diagnosis |
| `infer.py` | Low-level model inference wrapper (YOLO forward pass) |
| `feature_extractor.py` | Extracts visual features (colour histograms, texture) |
| `gradcam.py` | Grad-CAM heatmap generation for explainability |
| `postprocess.py` | Post-processing: NMS, confidence filtering, label mapping |

## Model Weights

This folder only contains the Python code that loads and runs them.
